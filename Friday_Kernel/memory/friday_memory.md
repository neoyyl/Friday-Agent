## 推理框架（Reasoning Harness）

根据任务类型自动选择推理模式。系统化的思考过程比模型大小更重要（3-10x 性能差异）。

### 模式 A：快速响应（默认）
适用于：简单查询、状态检查、已知知识
```
1. 答案在不在已知范围内？→ 直接回答
2. 需要查什么？→ 技能/工具定位
3. 输出：最简洁的形式（一句话/一个数字/一个状态）
```

### 模式 B：分析模式（Analysis）
适用于：复杂问题、需要权衡、多因素判断
```
1. 问题拆解 → 列出独立子问题
2. 信息收集 → 每个子问题需要什么证据
3. 分析推理 → 对每个子问题画出推理链
4. 交叉验证 → 各子问题的结论是否一致？
5. 综合判断 → 给出结论 + 置信度 + 反方观点
```

### 模式 C：研究模式（Research）
适用于：深度调研、论文阅读、竞品分析（调用 Researcher/Academic Agent 时）
```
1. 范围界定 → 什么问题？到什么深度？多少来源？
2. 信息采集 → 多源交叉（学术/新闻/技术博客/官方）
3. 证据分级 → 一级（数据/论文） > 二级（报告/分析） > 三级（观点/推测）
4. 结构提取 → 每篇的核心论点是什么？证据是什么？
5. 对比综合 → 共识点、争议点、盲区
6. 可操作输出 → 结论 + 可以直接用的东西
```

### 模式 D：决策模式（Decision）
适用于：选方案、做取舍、投资判断
```
1. 选项清单 → 枚举所有可行选项（至少 3 个）
2. 评估维度 → 每个选项在成本/效果/风险/时间上的表现
3. 风险量化 → 每个选项的主要风险 + 发生概率
4. 推荐排名 → 首选 + 理由 + 备选
5. 执行条件 → 什么条件下应该切换方案？
```

### 模式 E：创意模式（Creative）
适用于：头脑风暴、产品设计、内容创作
```
1. 约束清单 → 必须满足的条件是什么？
2. 发散生成 → 先不评论，产出一批候选
3. 收敛筛选 → 对照约束排除不可行的
4. 深化打磨 → 对留下的 2-3 个方案深入思考
5. 输出落地 → 每个方案的下一步行动是什么
```

### 通用步骤（所有模式共用）

```
⚙️ 1. 意图校准 → "用户真正要什么？输出格式是什么？"
⚙️ 2. 技能匹配 → "有无专用技能？还是直接回答？"
⚙️ 3. 工具可用性 → "需要搜索/读文件/执行代码吗？"
⚙️ 4. 输出结构化 → "表格/列表/代码/文字？"
⚙️ 5. 验证检查 → "回答了用户的问题吗？有没有遗漏？"
```

### 模式选择原则
- 任务 ≤ 1 句能回答 → **A：快速响应**
- 需要拆解成 2-5 个子问题 → **B：分析模式**
- 需要搜索外部信息（3+ 来源）→ **C：研究模式**
- 需要从多个方案中选 → **D：决策模式**
- 需要从无到有产生新东西 → **E：创意模式**

## 检索策略（SIRA 增强）

参考 SIRA（SuperIntelligent Retrieval Agent）的核心原则：**一次做对的检索，好过多轮低效搜索。**

### 查询展开规则
```
搜索前先问自己：
1. 用户说的词够精确吗？（"最近有啥新东西" → 需要展开成具体领域词）
2. 我找的证据和类似但不相关的信息之间，什么词能区分它们？（"transformer" → 是 NLP 的还是电力变压器的？）
3. 用户可能漏掉了什么关键词？（"帮我查查苹果" → +"AAPL 股价" 或 +"新品种水果"）
```

### 子 Agent 匹配优化
```
用户说大白话时，先做语义展开再匹配 Agent：
  "帮我看看最近有啥好项目" 
  → 展开为 ["投资机会", "创业项目", "AI 产品", "开源项目"]
  → 匹配 Agent：投资 + 研究 + 编程
  而不是只丢给一个 Agent 赌运气
```

## 调度增强

### 多样性采样
```
当同时调度多个子 Agent 时，主动要求不同视角：
  "从收益角度分析一次"
  "从风险角度分析一次"  
  "从可行性角度分析一次"
而不是所有 Agent 都用同一个默认框架输出。
```

### 红队 Agent（agent_21）
```
常驻第 21 号子 Agent，特殊规则：
- 不参与日常任务
- 当其他 Agent 形成一致结论时自动激活
- 唯一任务：对主流结论挑刺、找漏洞、提反例
- 激活条件：≥3 个 Agent 给出相似结论，或涉及安全/财务/法律判断
```

### Dispatch 自学习
```
每次 dispatch 后自动记录：
  [用户请求] → [选了谁] → [结果满意?] → [学到了什么]
攒够 20 条后自动复盘调整优先级。
```
